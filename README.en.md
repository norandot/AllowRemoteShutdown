# AllowRemoteShutdown (Remote Power Management Server & Web UI)

**AllowRemoteShutdown** is a lightweight, secure, and highly transparent remote power management server built with native Windows PowerShell (5.1) `HttpListener` and Microsoft Sysinternals `psshutdown.exe` (or Windows native `shutdown.exe`).

All backend logic is fully open-source and auditable, written completely in a single PowerShell script. This ensures perfect transparency for intermediate to advanced users who require robust security auditing within their private home networks.

---

## 🚀 Key Features

1. **Bilingual Support (EN / JP)**
   * Automatically detects the OS system locale (English/Japanese) at startup. It can also be overridden and locked to a specific language in the configuration.
2. **Comprehensive Power Modes**
   * Supports Shutdown, Restart, Log off, Suspend (Sleep), Hibernate, Monitor Off, and Lock Screen. These can be executed either from the system tray menu or via the modern Web UI.
3. **Seamless System Tray Integration**
   * Displays a clean, native tray icon using the Unicode symbol "⏼" (U+23FC) or custom `.ico` files. It provides real-time PID tracking, log window toggles, and parameter editing.
4. **Desync Prevention (Web Sync Protection)**
   * To prevent accidental execution when the host PC's quick-off settings are changed directly from the tray, the Web UI detects desynchronization. It blocks unauthorized actions until the user syncs the browser via the **⟳ (Reload)** button.
5. **Click-to-Abort & Encoding Fix**
   * When a shutdown command is remotely triggered, a balloon notification appears on the host machine. Clicking this notification or tapping "Abort" on the Web UI immediately executes `psshutdown -a` to cancel the countdown. Full UTF-8 URL encoding ensures message boxes are readable with no character corruption.
6. **3 Built-in Themes + Custom Skin Engine**
   * **Slate**: A clean, eye-friendly dark theme (Default).
   * **Terminal**: A retro command-line look with neon green monospace fonts.
   * **Minimal**: A flat, high-contrast, brutalist-style interface.
   * Also supports writing your own HTML/CSS custom layout (detailed in [SKIN_CUSTOMIZATION_GUIDE.md](./SKIN_CUSTOMIZATION_GUIDE.md)).
7. **Interactive Setup Wizard (`setup.ps1`)**
   * A wizard script that guides you through setting up port numbers, generating security tokens, configuring execution paths, and automatically registering EULA acceptance registry keys in one run.
8. **Minimal Blueprint (`LEARNING_MINIMAL_SERVER.ps1`)**
   * A clean, comment-heavy, minimal skeleton code highlighting HttpListener multi-threading and NotifyIcon integration, perfect for learners or developers looking to extend the codebase.

## 📦 Project File Structure

*   `AllowRemoteShutdown.ps1` - Main backend server script managing the HTTP listener, tray integration, and log writing.
*   `setup.ps1` - Interactive configuration wizard that generates `config.json` and prepares the environment.
*   `config.json` - Configuration file (automatically generated by `setup.ps1` or updated from the tray).
*   `LEARNING_MINIMAL_SERVER.ps1` - Simplified blueprint script designed for educational review.

## 🛠️ Installation & Setup

### Step 1: Run the Setup Wizard
Open PowerShell as **Administrator**, navigate to this folder, and run:

```powershell
Set-ExecutionPolicy RemoteSigned -Scope Process
.\setup.ps1
```
Follow the interactive prompts to define your Port (default: 3000), generate a security token, and set your PsShutdown executable path. This will automatically write `config.json` and register registry agreements to bypass the Sysinternals EULA prompt.

*Note: In respect of Microsoft Sysinternals licensing terms, this repository does not package or automatically download the `psshutdown.exe` binary. Please download it manually from the [Microsoft Sysinternals Official Page](https://learn.microsoft.com/en-us/sysinternals/downloads/psshutdown).*

### Step 2: Register URL ACL (for Non-Admin execution)
To allow the HTTP listener to bind to your desired port (e.g., 3000) under non-admin execution environments, run this command once as Administrator:

```powershell
netsh http add urlacl url=http://+:3000/ user=Everyone
```

### Step 3: Open Windows Firewall
Allow incoming TCP connections on your configured port (e.g., 3000) for devices within your local network:

```powershell
New-NetFirewallRule -DisplayName "AllowRemoteShutdown" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 3000 -Profile Private
```
*(For enhanced security, we strongly recommend restricting the source IP range to your LAN subnets).*

### Step 4: Create a Startup Shortcut
To launch the server invisibly in the background on startup:
1. Create a shortcut pointing to `AllowRemoteShutdown.ps1`.
2. Right-click -> **Properties**, and edit the **Target** field as follows:
   ```cmd
   powershell.exe -ExecutionPolicy Bypass -NoProfile -WindowStyle Hidden -File "C:\Path\To\AllowRemoteShutdown.ps1"
   ```
3. Click **Advanced** and check "Run as Administrator".
4. Copy this shortcut to the `shell:startup` folder.

## 📱 How to Use

1. **Start Server**: Launch the configured shortcut. The "⏼" icon will appear in your system tray.
2. **Access Web Client**: Navigate to `http://[PC_IP_Address]:[Port]/` from your smartphone or tablet.
   * If a security token is configured, access `http://[PC_IP_Address]:[Port]/?token=[Your_Token]` for the first time to store your credentials in the browser cookie.
3. **Trigger & Cancel**: Trigger your power action from the Web UI. If you need to cancel a countdown, click the balloon notification on the host computer or tap "Abort" on the Web UI.

## ⚙️ Configuration Properties (`config.json`)

| Option | Description | Default / Example |
| :--- | :--- | :--- |
| `port` | HTTP server binding port. | `3000` |
| `token` | Security access token (empty disables token checks). | `"random_generated_token"` |
| `psshutdownPath` | Full path to `psshutdown.exe`. If left empty, falls back to native `shutdown.exe` (with limited modes). | `"C:\PSTools\psshutdown.exe"` |
| `mode` | Default action mode for Quick Execution. | `"-s"` (shutdown) / `"-r"` (restart) etc. |
| `delay` | Default countdown delay in seconds for Quick Execution. | `15` |
| `skin` | Web layout skin style. | `"slate"` / `"terminal"` / `"minimal"` |

## ⚠️ Troubleshooting

* **Desynchronization Warning**: If you see "The PC's quick execution parameters were changed..." on the Web UI, it means your browser and server configurations are out of sync. Click **⟳ (Reload)** at the top of the page to synchronize, then try again.
* **Corrupted Characters**: PowerShell 5.1 requires a Byte Order Mark (BOM) to read UTF-8 files correctly. If you manually edit any `.ps1` file, make sure to save it using **"UTF-8 with BOM"** encoding.
* **Connection Refused**: Verify that your Windows Network Profile is set to **"Private"** instead of "Public", as public profiles reject inbound LAN traffic by default.
